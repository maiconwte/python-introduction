Módulos de Terceiros - Resumo
Módulos de terceiros são bibliotecas/pacotes desenvolvidos pela comunidade que não vêm com o Python por padrão. Você precisa instalá-los separadamente.

Módulos Populares:
Web/API: requests, fastapi, flask, django

Data Science: numpy, pandas, matplotlib, scikit-learn

Automação: selenium, beautifulsoup4

Banco de Dados: sqlalchemy, psycopg2, pymongo

Outros: openpyxl (Excel), Pillow (imagens), pygame (jogos)

Instalação Básica:
bash
pip install requests pandas numpy
Gerenciamento de Dependências - Guia Prático
1. Criar e Usar requirements.txt
Gerar arquivo com dependências atuais:
bash
# Salva TODAS as bibliotecas instaladas
pip freeze > requirements.txt

# Salva apenas as que seu projeto usa (recomendado)
# Instale primeiro: pip install pipreqs
pipreqs . --force
Conteúdo do requirements.txt:
txt
requests==2.31.0
pandas==2.1.4
numpy==1.24.3
fastapi==0.104.1
uvicorn[standard]==0.24.0
Instalar do requirements.txt:
bash
pip install -r requirements.txt
2. Ambiente Virtual com venv (Nativo do Python)
Criar e ativar ambiente:
bash
# 1. Criar ambiente
python -m venv venv

# 2. Ativar (Linux/Mac)
source venv/bin/activate

# 2. Ativar (Windows)
venv\Scripts\activate
Verificar e usar:
bash
# Verificar se está ativo (mostra o caminho do venv)
which python  # Linux/Mac
where python  # Windows

# Instalar dependências no ambiente
pip install requests pandas

# Salvar dependências
pip freeze > requirements.txt

# Desativar ambiente
deactivate
3. Ambiente Virtual com pyenv + virtualenv (Mais Avançado)
Instalação do pyenv:
bash
# Linux/Mac
curl https://pyenv.run | bash

# Adicionar ao shell (adicionar no ~/.bashrc ou ~/.zshrc):
export PYENV_ROOT="$HOME/.pyenv"
[[ -d $PYENV_ROOT/bin ]] && export PATH="$PYENV_ROOT/bin:$PATH"
eval "$(pyenv init -)"

# Windows (use o pyenv-win)
# https://github.com/pyenv-win/pyenv-win
Gerenciar versões do Python:
bash
# Listar versões disponíveis
pyenv install --list

# Instalar versão específica
pyenv install 3.11.5

# Listar versões instaladas
pyenv versions

# Definir versão global
pyenv global 3.11.5

# Definir versão local (para este projeto)
pyenv local 3.11.5
Criar ambiente virtual com pyenv-virtualenv:
bash
# 1. Instalar plugin (se ainda não tiver)
git clone https://github.com/pyenv/pyenv-virtualenv.git $(pyenv root)/plugins/pyenv-virtualenv

# 2. Criar ambiente vinculado a uma versão do Python
pyenv virtualenv 3.11.5 meu-projeto-env

# 3. Ativar ambiente
pyenv activate meu-projeto-env

# 4. Trabalhar no projeto (instalar dependências, etc.)
pip install requests pandas
pip freeze > requirements.txt

# 5. Desativar
pyenv deactivate
4. Workflow Completo - Do Zero ao Projeto
Passo a Passo:
bash
# 1. Criar projeto
mkdir meu-projeto
cd meu-projeto

# 2. Criar ambiente virtual (escolha um método)
python -m venv venv  # ou
pyenv virtualenv 3.11.5 meu-projeto-env

# 3. Ativar ambiente
source venv/bin/activate  # ou
pyenv activate meu-projeto-env

# 4. Instalar dependências
pip install requests pandas fastapi

# 5. Criar requirements.txt
pip freeze > requirements.txt

# 6. Criar arquivo .python-version (se usar pyenv)
echo "3.11.5" > .python-version

# 7. Criar .gitignore
echo "venv/
__pycache__/
*.pyc
.env" > .gitignore

# 8. Codificar seu projeto...

5. Dicas Importantes
Manter requirements.txt atualizado:
bash
# Atualizar manualmente
pip install --upgrade package_name
pip freeze > requirements.txt

# Instalar versão específica
pip install package==1.2.3

# Instalar sem cache (problemas de versão)
pip install --no-cache-dir -r requirements.txt
Verificar dependências:
bash
# Ver dependências instaladas
pip list

# Ver dependências de um pacote
pip show package_name

# Verificar conflitos
pip check
Desinstalar tudo:
bash
# Lista e remove todas
pip freeze | xargs pip uninstall -y
Resumo dos Comandos Essenciais
bash
# Ambiente nativo (venv)
python -m venv venv
source venv/bin/activate      # Ativar
deactivate                    # Desativar

# pyenv
pyenv install 3.11.5          # Instalar Python
pyenv virtualenv 3.11.5 env   # Criar ambiente
pyenv activate env            # Ativar
pyenv deactivate              # Desativar

# pip
pip install package           # Instalar
pip install -r reqs.txt       # Instalar do arquivo
pip freeze > requirements.txt # Salvar dependências
pip list                      # Listar instalados
O seguro é sempre usar ambiente virtual e versionar o requirements.txt no Git!
